Переменная интерфейсного типа содержит в себе информацию о 2 типах: 
- статический тип -  тип самого интерфейса
- динамический тип - непосредственно тип, который реализует этот интерфейс

Под капотом переменная интерфейсного типа будет представлять из себя следующую структуру:
```
type iface struct {
    tab  *itab
    data unsafe.Pointer
}
```
- tab - указатель на структуру itab 
- data - указатель на данные (значение) динамического типа  

Структура itab содержит следующую информацию о статическом и динамическом типах:
```
type itab struct {
    inter  *interfacetype
    _type *_type
    hash uint32
    _ [4]byte
    fun [1]uintptr
}
```
- inter - метаданные интерфейса (статический тип)
- _type - указатель на динамический тип значения в интерфейсе
- hash - хэш, вычисленный на этапе компиляции для указанного _type (для последующего быстрого type assert/type switch, а также быстро помогает подтвердить тип, если мы присваиваем интерфейс какому-то типу)
- _ - поле для выравнивания структуры в памяти
- fun - указатель на header массива, где будут храниться методы динамического типа, которые удовлетворяют интерфейсу

В случае пустого интерфейса:
поскольку у нас нет никакой информации о интерфейсе (его методах и прочих метаданных), то переменная такого интерфейсного типа будет содержать только данные data

Текущая задача:
```var err *os.PathError``` - указатель на структуру ```os.PathError```, которому мы присвоили значение nil
Данная структура по умолчанию реализует интерфейс error (то есть реализует все методы, объявленные в этом интерфейсе):

```
type error interface {
    Error() string
}
```

Поскольку ф-ия возвращает интерфейс error, то при вызове ```return err``` происходит неявное преобразование конкретного типа в интерфейсный тип
В итоге в интерфейсном типе: 
- в структуре```itab``` в поле ```_type``` будет хранится информация о динамическом типе ```PathError```
- а в data в структуре ```iface``` положится текущее значения указанного типа -> nil


**ф-ия main:**
```
fmt.Println(err)
```
Вывод: nil, поскольку текущее значение динамического типа = nil
```fmt.Println(err == nil)```
Но поскольку в переменной интерфейсного типа хранится информация о динамическом типе (PathError), то сравнение с nil выведет false
