Вывод: error
Объяснение: 
у нас есть структура customError, реализующая метод Error
в сигнатуре ф-ии test объявлено, что она возращает указатель на данную структуру. При вызове return в значении данной структруры будет храниться nil
в ф-ии main мы объявляем переменную интерфейсного типа err. При выполнении присваивания err = test() в качестве информации о динамическом типе
будет записана информация о customError, а в качестве значения данного типа - nil. Поскольку переменная интерфейсного типа содержит информацию о динамическом типе (типе, реализующем интерфейс), то err != nil, а значит выведется сообщения об ошибке.
